Crear un nuevo proyecto con vue cli

Al ejecutar el siguiente comando se crea un nuevo proyecto con las indicaciones de configuración personalizadas

    vue create <nombr del proyecto>
    > Manually select features 
        > Babel
        > Router
        > Vuex
        > Linter / Formater 
    > Pick linter / formater config 
        > ESLint + Prettier 
    > Pick additional lint features : 
        > Lint on save 
    > Where do you preferer placing config for Babel, PosCSS, ESLint 
        > In dedicated config file 
    # Esta opción permite guardar la configuración anterior para futuros proyectos 
    > Save this as a preset for future projects : N

Carga perezosa en vue cli

En el router es recomendado cargar bajo demana los componentes




 
 






















    # src/router.js
    import Vue from 'vue';
    import Router from 'vue-router';
    const Home   = () => import('./views/Home.vue');
    const About  = () => import('./views/About.vue');


    Vue.use(Router);

    export default new Router({
    mode: 'history',
    base: process.env.BASE_URL,
    routes: [
      {
        path: '/',
        name: 'home',
        component: Home,
      },
      {
        path: '/about',
        name: 'about',
        component: About,
      },
    ],
   });

Nota

Es recomendable instalar la libreria de babel para importar dinámicamente + npm install --save-dev @babel/plugin-syntax-dynamic-import + Colocar el plugin en el archivo .babelrc { "plugins": ["@babel/plugin-syntax-dynamic-import"] }

Si se quiere agrupar varios componentes en un solo paquete de webpack se puede incluir un nombre del paquete

const Foo = () => import(/* webpackChunkName: "group-foo" */ './Foo.vue')
const Bar = () => import(/* webpackChunkName: "group-foo" */ './Bar.vue')
const Baz = () => import(/* webpackChunkName: "group-foo" */ './Baz.vue')
